# Brolly

> The forecast, without the noise.

Brolly is an ad-free, tracking-free weather and radar app for the UK (with
global fallback). It replicates the core experience of "Weather & Radar"
— animated precipitation radar, current conditions, multi-day forecast,
and Met Office severe weather warnings — without any ads, analytics,
accounts, or paywalls.

- **Forecasts:** Met Office DataHub (primary, UK) + Open-Meteo (fallback / global)
- **Radar:** RainViewer animated precipitation tiles (no key required)
- **Storage:** drift (SQLite) for saved locations, shared_preferences for settings
- **State:** Riverpod
- **Map:** flutter_map over OpenStreetMap base tiles

## First-time setup

This repo only contains the cross-platform `lib/` code and configuration —
the Android / iOS native scaffolds are generated by the Flutter tooling.

### 1. Get a Met Office DataHub API key (optional but recommended)

1. Sign up at <https://datahub.metoffice.gov.uk>.
2. Create an application and subscribe it to:
   - **Site Specific Forecast API**
   - **Severe Weather Warnings API**
3. Copy the API key.

The free tier is 360 calls/day per API. If you skip this step the app will
still work using Open-Meteo as the forecast source — severe weather
warnings will be hidden.

### 2. Create your `.env`

```bash
cp .env.example .env
# then edit .env and paste your key after `MET_OFFICE_API_KEY=`
```

`.env` is gitignored — it never gets committed.

### 3. Generate the platform folders

```bash
# Run from the repo root. Replace `yourname` with whatever you'd like —
# this becomes the Android package id (e.g. com.alice.brolly).
flutter create . --org com.yourname --project-name brolly --platforms=android,ios
```

`flutter create` won't overwrite your `lib/` or `pubspec.yaml`. After it
finishes, merge the manifest fragments shipped in the repo:

- Add the `<uses-permission>` lines from
  `android/app/src/main/AndroidManifest.xml.fragment` into the generated
  `android/app/src/main/AndroidManifest.xml` (just above the
  `<application>` tag).
- Add the `NSLocationWhenInUseUsageDescription` / `…AlwaysAndWhenInUse…`
  keys from `ios/Runner/Info.plist.fragment` into the generated
  `ios/Runner/Info.plist`.

### 4. Install dependencies and run code generation

```bash
flutter pub get
flutter pub run build_runner build --delete-conflicting-outputs
```

The code-gen step creates the `.g.dart` files for Retrofit, json_serializable,
and Drift. They're gitignored — anyone cloning the repo needs to run this once.

### 5. (Optional) Branding & splash

Drop the three PNGs described in [`assets/branding/README.md`](assets/branding/README.md),
then:

```bash
flutter pub run flutter_launcher_icons
flutter pub run flutter_native_splash:create
```

### 6. Run

```bash
flutter run                  # auto-detects a connected device
flutter run -d <device-id>   # or pick one
```

## Project layout

```
lib/
  main.dart                       # bootstraps dotenv, SharedPreferences, ProviderScope
  app.dart                        # MaterialApp.router
  core/
    env/env.dart                  # .env accessor
    router/app_router.dart        # go_router + bottom navigation shell

    settings/                     # Units + theme + saved-locations management
  shared/widgets/                 # Loading + error views
```

## How the Met Office → Open-Meteo fallback works

See `lib/data/repositories/weather_repository.dart`. In short:

1. If no Met Office key is in `.env`, the app always uses Open-Meteo.
2. If the requested point is outside the UK bounding box, the app uses
   Open-Meteo (Met Office only covers the UK).
3. Otherwise Met Office is tried first. A 429 (quota exhausted) or
   401 / 403 (bad key) response flips an in-memory disable flag for the
   rest of the UTC day and immediately retries the same call against
   Open-Meteo. Any other Met Office error bubbles up — we don't want a
   transient blip to silently switch providers forever.

The `WeatherForecast` domain model carries a `source` field so the UI can
show which provider answered.

## Privacy

- No ads SDKs, no analytics, no Crashlytics, no Firebase, no tracking pixels.
- No account or login.
- Location stays on-device — it's only sent to forecast providers as
  `latitude` / `longitude` query parameters.
- The Met Office API key lives in `.env` and is read with `flutter_dotenv`.

## Non-goals (v1)

- No push notifications
- No tablet-specific layouts
- No home-screen widgets (separate project)
- No paid tiers or subscriptions — ever
